# PLAN-002 — self-reflection session method

**Source plan:** `.orchestrator/decisions/SELF-REFLECT-2026-07/plan-response.txt` (authoritative except
where amended below).

## Claude's challenge
1. **Bloat-recommending-bloat:** 9 artifacts, ~6 fresh-context roles, a rebuttal round, and stratified
   sampling — for a repo with ~16 specs and ~1 week of history. The session design partially trips its
   own named failure mode.
2. **Misassigned work:** "the operator freezes scope and writes 00-charter.md" gives Val work he
   delegated; the operator's role is end-stage adjudication only.
3. **Provenance theater:** per-file checksums + tool versions for a same-box, same-day audit duplicate
   what the frozen commit SHA already proves. Hashing is load-bearing ONLY for the two blind reports
   (pre-disclosure commitment).
4. **Missing evidence class:** attempt records cover pipeline tasks but not ORCHESTRATION-LEVEL
   operational efficiency (consult round-trips, incidents like the codex stdin hang, the
   `--delete-branch` bug, ProtectHome/AppArmor dead-ends). Auditors need a neutral incident log.

## Dispositions
1. Collapse roles: 2 blind auditors (SOL + fresh-context Claude) → Claude-side reconciliation matrix →
   ONE SOL verification pass of the reconciliation. Rebuttal round ONLY if the matrix contains a
   factual (not value) dispute. Artifacts trimmed to: charter+manifest (one file), baseline metrics,
   rubric, 2 blind reports (+hashes), reconciliation, verified final findings + capped action list.
2. Claude writes the charter as delegate; Val adjudicates unresolved value judgments at the end.
3. Freeze = integration commit SHA in the charter; sha256 ONLY the two blind reports before disclosure.
4. A neutral operational-incident log (facts only, no analysis) is added to the evidence bundle; the
   rubric gains a "session/operational efficiency" input.

## Authorized unamended
Frozen-commit evaluation; docs=intent vs code/evidence=behavior; two fully blind vendor passes with the
SAME rubric; the four adversarial frames ("overengineered", "under-controlled", "smallest system
preserving demonstrated protections", "claims supported only by our own documentation"); fact vs
inference vs recommendation labeling; praise only when tied to a demonstrated prevented failure; the 7
audit dimensions + deletion test; keep/fix/simplify/remove with severity/confidence/citations/
counterevidence/smallest-viable-action; deterministic baseline from dispatch metrics +
delegation_report; final action list CAPPED AT 10, each with acceptance test + net complexity change;
ceremony-adding recommendations must name what they replace.

## Validation
Complete when: both blind reports exist and were hashed pre-disclosure; all 7 dimensions covered in
both; every material claim cites path/commit/attempt; disagreements resolved or explicitly open for
Val; ≤10 actions each with a falsifiable completion test.
